# Definition of Done

A change is complete when it meets the following criteria:

- Code changes are small and focused.
- Behavior is preserved unless intentionally changed.
- Imports are explicit and package-aware.
- Configuration changes are documented.
- The backend starts successfully.
- The frontend starts successfully.
- Database initialization is verified.
- Health endpoint works.
- Swagger or API docs load successfully.
- Frontend-to-backend communication is verified for relevant features.
- Tests are added or updated when needed.
- Documentation is updated for developer experience changes.
- The change is reviewable and maintainable.
